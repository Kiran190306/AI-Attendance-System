 HOW TO DEPLOY (10 MINUTES)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STEP 1: DEPLOY BACKEND (2-3 minutes)
────────────────────────────────────

a) Push to GitHub:
   $ git add .
   $ git commit -m "Production deployment"
   $ git push origin main

b) Create Render Web Service:
   - Go to https://dashboard.render.com
   - Click "New +" → "Web Service"
   - Connect your GitHub repository
   - Configure:
     * Build: pip install -r backend/requirements.txt
     * Start: gunicorn -w 4 -b 0.0.0.0:$PORT backend.app:app
     * Environment Variables:
       - SECRET_KEY: <generate 32-character random string>
       - DEBUG: False
   - Click "Create Web Service"

c) Wait for deployment to complete

d) Copy your Render URL (e.g., https://ai-attendance-backend.onrender.com)


STEP 2: BUILD ANDROID APP (3-5 minutes)
───────────────────────────────────────

a) Update Android Configuration:
   File: android_app/app/src/main/java/com/attendance/app/api/RetrofitClient.java
   
   Find:    private static final String BASE_URL = "https://your-render-url.onrender.com";
   Replace: private static final String BASE_URL = "https://YOUR-ACTUAL-RENDER-URL";

b) Build Debug APK:
   $ cd android_app
   $ ./gradlew assembleDebug

c) Install on Device:
   $ adb install app/build/outputs/apk/debug/app-debug.apk

d) Or run directly from Android Studio:
   - Click Run (green play button)
   - Select your device/emulator


STEP 3: TEST THE SYSTEM (2 minutes)
──────────────────────────────────

a) Login Screen:
   - Username: demo
   - Password: password123
   - Click "Login"

b) Dashboard:
   - View analytics cards
   - Pull down to refresh
   - Click "View Attendance"

c) Attendance Screen:
   - See attendance records
   - Pull down to refresh
   - Back to Dashboard

d) Logout:
   - Click "Logout" button
   - Should return to login

✅ DONE! System deployed and working!

 BACKEND API ENDPOINTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. POST /api/login
   Request:  {"username":"demo", "password":"password123"}
   Response: {"access_token":"...", "token_type":"bearer", "user":{...}}
   Use: User authentication, get JWT token

2. POST /api/signup
   Request:  {"username":"user", "email":"user@example.com", "password":"pass"}
   Response: {"access_token":"...", "token_type":"bearer", "user":{...}}
   Use: New user registration

3. GET /api/attendance (Protected with JWT)
   Header:   Authorization: Bearer <token>
   Response: [{"student_name":"...", "date":"...", "timestamp":"...", ...}]
   Use: Retrieve attendance records

4. GET /api/analytics (Protected with JWT)
   Header:   Authorization: Bearer <token>
   Response: {"total_students":50, "present_today":48, "attendance_percentage":96.0, ...}
   Use: Get analytics data
